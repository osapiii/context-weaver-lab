# StoryVault

StoryVaultは、画面操作を記録した動画からユーザーストーリーと根拠情報を抽出し、プロダクト開発のコンテキストとして再利用できるようにするアプリケーションです。

AIによって実装速度が大きく向上した一方で、「何のために作った機能なのか」「本当にユーザー価値につながっているのか」という背景は失われやすくなっています。StoryVaultは、実際の操作、説明、画面、関連資料をユーザーストーリーとして整理し、人とAIの双方が参照できる形で蓄積します。

## 主な機能

- 操作動画の録画とアップロード
- 無音区間やノイズを考慮した動画解析
- 操作内容ごとのクリップ分割
- 発話の文字起こしと代表画面の抽出
- `Who / What / Why` と完了条件を持つユーザーストーリーの生成
- ユーザーストーリーと動画・発話・画面の根拠情報の関連付け
- 関連するチケット、コード、Pull Request、ドキュメントの整理
- コンテキストマップによる機能間の関係の可視化
- AIツールから蓄積情報を取得するためのMCP連携
- ユーザーマニュアルやプロジェクト報告書の生成

## 利用イメージ

1. 対象のアプリケーションを選択して、画面を操作しながら説明します。
2. StoryVaultが動画を解析し、操作のまとまりごとに整理します。
3. 操作と説明からユーザーストーリー、完了条件、根拠情報を生成します。
4. チームメンバーやAIツールが、目的に応じて必要なコンテキストを参照します。

## 活用例

| 利用者 | 活用方法 | 成果物 |
|---|---|---|
| プロダクトマネージャー | 実装済み機能とユーザー価値を整理する | バックログ、進捗報告書、リリース計画 |
| エンジニア | 操作動画と仕様を確認しながら実装・修正する | 実装コード、Pull Request、技術調査メモ |
| QA・レビュアー | 完了条件と実際の動作を照合する | テストケース、レビューコメント、不具合報告 |
| デザイナー | 操作の流れと画面の意図を確認する | UIレビュー、改善案 |
| サポート担当者 | 実画面と操作手順を再利用する | 操作マニュアル、FAQ、案内資料 |
| プロジェクトマネージャー | 根拠付きで実装状況を把握する | 週次報告、進捗一覧、リスク管理表 |

## ユーザーストーリーの考え方

StoryVaultは、機能名だけではなく、次の情報を一つの単位として扱います。

- **Who**: 誰のための機能か
- **What**: 何ができるようになるのか
- **Why**: なぜそれが必要なのか
- **完了条件**: どの状態になれば価値を提供できたと判断できるか

機能を追加するコストが下がった今だからこそ、実装量ではなく「チームが本当に価値のある機能を提供できているか」を確認するための共通言語として活用できます。

## 技術要素

本アプリケーションは、Google Cloud、Firebase、Gemini、Agent Development Kit、Model Context Protocolなどを活用して構築しています。
