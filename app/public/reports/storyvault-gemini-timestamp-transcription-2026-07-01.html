Gemini タイムスタンプ付き文字起こし移行レポート

Aqua Voice の実 API では timestamped JSON / SRT が返らなかったため、StoryVault のザッピング動画保存フローを Gemini / Vertex AI によるタイムスタンプ付き文字起こしへ切り替えました。 ストーリー解析は transcriptSegments を第一情報源とし、根拠 cue のない story/evidence は受け付けません。

確認した API 仕様

Aqua Voice

verbose_json、timestamp_granularities[]=segment、timestamps=true 等を実 API で確認しましたが、レスポンスは text / usage / _request_id のみでした。response_format=srt は 400 でした。

Gemini / Vertex AI

audio/wav を gemini-2.5-flash に渡し、segments[{startMs,endMs,text}] 形式の JSON が返ることを実 API で確認しました。

実装内容

  • Firebase callable transcribe_zapping_video_with_aqua の中身を Gemini timestamp transcription に置換しました。関数名は既存フロント配線維持のため残しています。
  • 録画保存時に timingStatus === "timestamped"、transcriptSegments.length > 0、transcriptSrt を必須化しました。
  • ストーリー解析開始時も、全 clip が timestamped transcript を持たない場合は事前に停止します。
  • ADK の prompt/schema/tools から legacy fallback を削除し、evidence.transcriptCueIds と evidence.transcriptQuote を必須にしました。
  • SRT だけが渡った場合も ADK tools で cue ID 付き transcriptSegments に正規化します。
