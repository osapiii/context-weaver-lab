ANAI Studio の動画エディターを StoryVault へ移植

en-aistudio の Video Studio 実装を参照し、StoryVault の動画詳細タブから音声付き・字幕付き動画生成フローを扱えるようにしました。 Cloud Run の Vohance 系マイクロサービスと Firebase Functions トリガーも dev 環境へ展開済みです。

実装サマリー

UI 移植 StoryVault の動画詳細に 動画生成 タブを追加し、ANAI Studio の VideoStudioWorkspace をそのまま利用する構成にしました。
既存動画の取り込み StoryVault の撮影済み動画を Video Studio の動画ドキュメントとして登録する bridge パネルを追加しました。
バックエンド移植 Vohance Cloud Run サービス群、Firestore request trigger、Request Log registry を追加しました。

主な追加・変更ファイル

  • app/components/storyVault/StoryVaultOperationVideoPanel.vue: 詳細タブに 動画生成 を追加。
  • app/components/storyVault/StoryVaultVideoGenerationPanel.vue: StoryVault の動画を Video Studio に接続する bridge。
  • app/components/AgentWorkspace/VideoStudioWorkspace.vue: ANAI Studio 由来の動画編集 UI 本体。
  • app/stores/videoStudio.ts, app/types/models/videoStudio.ts, app/utils/videoStudioStoragePaths.ts: Video Studio のデータモデルと保存ロジック。
  • backend/app/triggers/vohance_video_requests.py: 動画生成 request を Cloud Run に中継する Firebase Functions。
  • backend/microservice/vohance/: 動画分割、文字起こし、TTS、音声合成、字幕追加などの Cloud Run サービス。
  • scripts/deploy-vohance-video-workers.sh: StoryVault dev project 向けの Cloud Run デプロイスクリプト。

利用フロー仕様

StoryVault の動画詳細画面で 動画生成 タブを開くと、現在の操作動画を Video Studio の編集対象として初期化します。 以後は ANAI Studio と同じ UI でセクション分割、文字起こし、ナレーション生成、字幕付与、最終動画の結合を実行できます。

1. 撮影済み動画 StoryVault の operation video / clip をソースとして参照。
2. Bridge 登録 createStorageBackedVideo が GCS 既存動画を Video Studio doc に紐付け。
3. Editor UI VideoStudioWorkspace で字幕・音声・セクションを操作。
4. Request Docs Firestore の requests/*/logs に各処理 request を作成。
5. Cloud Run Firebase Functions が Vohance services を呼び、成果物を Storage / Firestore に反映。
