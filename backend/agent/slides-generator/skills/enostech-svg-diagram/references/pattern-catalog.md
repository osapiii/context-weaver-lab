# SVG パターンカタログ

「どんな絵を描きたい時にこのスキルを選ぶか」の判断材料。
題材ごとに「向いているか / 別スキル (DIAG / CHART) の方が向くか」を整理した。

## SVG が圧倒的に向く題材

| 題材 | なぜ SVG か | 描き方の核 |
|------|-----------|-----------|
| データパイプライン | 各ノードを「絵」で示せる (歯車・キュー・円筒) | rect + Q 曲線 + 並んだ箱 |
| 電子回路 / 物理装置 | path で正確な装置形状を描ける | 矩形 + 直線 + 三角 |
| メカニズム図 (歯車連動) | 円 + 線で「動き」が出る | 歯車レシピ (how-to-write-svg.md コツ3) |
| ダッシュボード模式図 | 折れ線・棒グラフを内側に直接描ける | ミニグラフ レシピ (コツ5) |
| センサー / IoT 端末 | アンテナ波 (Q 曲線) で「発信」を表現 | アンテナ波レシピ (コツ1) |
| 信号フロー / データ流路 | 矢印 + 点線で多層フロー | marker 1 種 + dasharray 4 4 |

## SVG でも描けるが DIAG (シェイプ) でも十分な題材

これらは「PowerPoint 上で文字を直したい」ニーズが強ければ DIAG を使う:

| 題材 | DIAG 候補 |
|------|----------|
| サイクル図 (PDCA 等) | DIAG-02 |
| Before/After 比較 | DIAG-04 |
| ピラミッド (3-5 階層) | DIAG-05 |
| タイムライン | DIAG-06 |
| 2x2 マトリクス | DIAG-08 |
| FlowChart (判断木) | SCENE-06 (ENO-85 と組み合わせ) |
| 階層型組織図 | (DIAG にはまだ無い、SVG 推奨) |

## SVG では描かない題材

| 題材 | 使うべきもの |
|------|------------|
| 棒グラフ・折れ線・円グラフ等の定量データ | CHART-01〜09 (enostech-slides) |
| 3 列以上の表データ | ENO-13 / ENO-30 / ENO-48 (テーブル) |
| 写真・実物画像 | ENO-87 (リファレンス画像) or 別途画像挿入 |

## 判断フロー

```
図を入れたい
  ↓
定量データ (数字の比較) か?
  → YES: CHART-01〜09 を使う (このスキルは使わない)
  → NO: ↓

行と列の表形式情報か?
  → YES: ENO-13 / ENO-30 / ENO-48 (このスキルは使わない)
  → NO: ↓

絵としての説得力が要るか? (歯車・キュー・回路・装置・ダッシュボード等)
  → YES: SVG (このスキル) ← デフォルト
  → 微妙 / 単純な構造図: ↓

ユーザーが「PowerPoint で文字を直したい」と明示しているか?
  → YES: DIAG-02〜09 / SCENE-01〜06 (enostech-slides)
  → NO: SVG (このスキル) ← デフォルト
```

VISUAL 系スライドの図解は、迷ったら SVG (このスキル)。

## SECSUMMARY-1 専用パターン: 章扉直後の主役ビジュアル一発 (v9.4 確定仕様)

enostech-slides v9.4 で SECSUMMARY-1 は **「章扉直後に主役ビジュアル一発を見せる
full-bleed SVG-only template」** と再定義された。

### 絶対ルール

- **画面 (10" × 5.625") の 100% を SVG が占有する**
- **SVG = 主役ビジュアルのみ** (タイトル / サブ / 章 chips / 結論バー / 章番号などの装飾は一切書かない)
- テンプレ側も chrome (header / footer / left strip / page number) を描かない

### 役割分担表

| 要素 | テンプレが描く | SVG が描く |
|---|---|---|
| 上部 amber 帯 | ❌ | ❌ (描かない) |
| 章タイトル「第 NN 章 — TITLE」 | ❌ | ❌ (描かない) |
| サブタイトル one_line | ❌ | ❌ (描かない) |
| 章ナビ chip リボン (全章 chip) | ❌ | ❌ (描かない) |
| 結論バー (下部の amber 帯) | ❌ | ❌ (描かない) |
| **中央の主役ビジュアル (構造図 / 関係図 / メタファー)** | ❌ | ✅ (これだけ) |
| 下部 chrome (ロゴ / ページ番号) | ❌ | ❌ (描かない) |

### なぜ「主役ビジュアル一発のみ」か

章扉 (SECTION-2/4/5) で章タイトルと方向性は既に提示されている。SECSUMMARY-1 はその直後に
**1 枚絵で「これから何の話をするのか」を視覚的に渡す**役割。タイトルや章番号や全章 chips を
重ねて描くと、読者の視線が「絵」ではなく「文字情報」に取られ、見取り図としての効果が薄れる。
v9.4 で「ビジュアル一発」に振り切ることで、章扉と SECSUMMARY-1 の役割分担が明確になる。

### SVG が描く中央ビジュアルの規範

```
viewBox="0 0 1920 1080"  (16:9、画面いっぱいに占有)
canvas:  <rect width="1920" height="1080" fill="#FAFAF7"/>  (warm-bg)

主役構造 (1 つだけ):
  例 1) 概念モデル: 中央に親 box (#FFFFFF / #374151) + 子 boxes 横並び (#FEF3C7 + amber stroke)
  例 2) フロー: 縦/横の amber stroke ステップカード 3〜5 個 + 接続線 (#1F2937, stroke-width 3)
  例 3) 関係図: 中央メタファー + 周辺 3〜4 要素を直線/Q曲線で接続
  例 4) 比較: 左右 2 ブロック (片方を amber 強調)

色の使い方:
  - canvas: warm-bg #FAFAF7
  - 強調塗り: dark gray #374151 (1 箇所のみ、主役の主役) + 文字 amber #F59E0B
  - カード塗り: light yellow #FEF3C7 + amber stroke #F59E0B (3〜5 個まで)
  - ink #1F2937 は塗りに使わず、文字色のみ
  - 線色: ink #1F2937 (stroke-width 2 or 3) または amber stroke
```

### NG パターン (やってはいけない)

- ❌ SVG 内に「第 01 章 — タイトル」のテキストを書く (主役ビジュアルではない装飾)
- ❌ SVG 内に章サブタイトル (one_line) を書く
- ❌ SVG 内に全章 chips ナビ (01 / 02 / 03 ...) を書く
- ❌ 下部に結論バー / 締めのメッセージ帯を置く (主役ビジュアルから視線が離れる)
- ❌ ink (`#1F2937`) で塗る面を作る (R-SVG-12 fatal)
- ❌ amber を 4 箇所以上で「塗る」(R-SVG-9 fatal、塗りは 15% 以下)
- ❌ contain で余白が出るような小さい構造を中央に置く (主役は画面いっぱい大きく描く)

### OK パターン

- ✅ 主役構造を 1 つだけ画面中央に大きく描く (viewBox の 70% 以上を主役で埋める)
- ✅ amber は **stroke** で使い、塗りは light yellow `#FEF3C7` に逃がす
- ✅ 強調 1 箇所だけ dark gray (`#374151`) 塗り + amber 文字
- ✅ カード数は 3〜5 個 (それ以上は LIST-3 系を検討すべき情報量)
- ✅ 各要素のラベル (短い名前・概念名) は書く (主役ビジュアルの内訳として)

### plan.json での書き方 (Phase 2 LLM 向け)

```jsonc
{
  "id": "S6",
  "template_id": "SECSUMMARY-1",
  // 以下 3 つは speaker notes / 設計メモ用 (画面には出ない)
  "section_no":     "01",
  "section_title":  "なぜ Semantic Layer が必要か",
  "one_line":       "同じ MAU が 3 通り — Flowtide 事件の構図と SL の解決構造",
  // 主役ビジュアル一発の SVG (画面いっぱいに描かれる)
  "svg": "<svg viewBox='0 0 1920 1080' ...>主役構造のみ</svg>"
}
```

## カード背景配色マトリクス (v1.8 / R-SVG-16 強制対象)

SECSUMMARY-1 / SVG 図解で「テキストカード / コールアウト」の **背景塗り** を決める時の
判断基準。**amber 系塗り (#FCD34D 〜 #B45309) でカード背景は禁止** (R-SVG-16 fatal)。

### NG パターン (機械検出される / 修正必須)

```
┌──────────────────────────┐  ← <rect width="500" height="84" fill="#F59E0B"/>
│  通常シューズ — 守る設計  │  ← 内側に <text> = R-SVG-16 fatal
└──────────────────────────┘
```

直近のベアフットデッキ SECSUMMARY-1 (S6 / S12 / S19 / S26) で採用していた、
カード上端 74〜100px に amber rect を載せて白文字でタイトルを書く構図。これは
osanai さんが「ダサい」と指摘した代表例で、v1.8 で fatal 化した。

### OK パターン (推奨カード配色 4 系統)

| 用途 | 背景 | stroke | headline 文字色 | 用例 |
|---|---|---|---|---|
| **白カード + amber アクセント** (汎用 デフォルト) | `#FFFFFF` | `#F59E0B` 3px / 左 4px stripe のみ | `#B45309` (amber-700) | ステージカード、比較カード、判定木の Q ボックス |
| **薄黄カード** (柔らかい強調) | `#FEF3C7` (amber-100) | `#F59E0B` 2-3px | `#B45309` | 主役の主役 (1〜2 箇所) |
| **薄グレーカード** (中立トーン) | `#F5F5F4` 〜 `#E5E7EB` | `#9CA3AF` 2px | `#374151` (ink) | 補助情報、参考情報 |
| **dark inverse カード** (1 枚だけ強調) | `#1F2937` 〜 `#374151` | (枠線なし) | `#FFFFFF` 本文 + `#FBBF24` 強調語 | 「結論」「核」「違いはここ」を 1 枚 |

### テーマ別 推奨組み合わせ

| テーマ | 推奨レイアウト | NG 例 |
|---|---|---|
| **ステージ / 段階 (3-5 個)** | 白カード × N、進行に従って 1 つだけ dark inverse で強調 | 全カード amber 塗り |
| **比較 (2 ブロック)** | 白カード + amber stroke を左右、片方だけ dark inverse で「核」を強調 | 両方とも amber header 帯 |
| **タイムライン (横軸)** | ベース線 1 本 (`#9CA3AF`) + ノード ●、節目だけ amber 文字色で強調 | 各ノード box に amber 塗り |
| **判定木 (Q ボックス)** | 薄紫枠ボックス (`#EDE9FE` + `#9212F3` stroke 2px) で Q、Yes/No 線分岐 | Q ボックス自体を amber 塗り |
| **カタログ / 一覧 (4-8 個)** | 白カード + 細い amber stripe (左 4px) | 全カード薄amber 塗り (区別がなくなる) |

### amber を「使ってよい」用法 (R-SVG-16 で除外される使い方)

- **stroke** (枠線・罫線・矢印) — 太さ 2 or 3px
- **文字色** (headline, 強調語, ラベル, アイコン文字)
- **薄帯** (高さ < 50px の区切り罫線・section divider・アンダーライン)
- **アイコン fill** (40px 角未満 / 面積 1600 px² 未満の小シンボル・badge・dot)
- **light yellow #FEF3C7 (amber-100) は OK** — amber-200 以降と区別して、薄い背景には使ってよい

### 移行レシピ (既存の amber 塗りカードを直す)

```diff
- <rect x="990" y="160" width="760" height="84" fill="#F59E0B" rx="18"/>
- <text x="1370" y="216" font-size="34" fill="#FFFFFF">ベアフット — 感じる設計</text>
+ <rect x="990" y="160" width="760" height="84" fill="#FFFFFF" stroke="#F59E0B"
+       stroke-width="3" rx="18"/>
+ <rect x="990" y="160" width="6" height="84" fill="#F59E0B"/>  <!-- 左 4-6px stripe -->
+ <text x="1370" y="216" font-size="34" fill="#B45309" font-weight="700">ベアフット — 感じる設計</text>
```

または「核を強調したい 1 枚」だけ:

```diff
- <rect x="990" y="160" width="760" height="84" fill="#F59E0B" rx="18"/>
- <text x="1370" y="216" font-size="34" fill="#FFFFFF">核となる要素</text>
+ <rect x="990" y="160" width="760" height="84" fill="#374151" rx="18"/>
+ <text x="1370" y="216" font-size="34" fill="#FFFFFF">核となる要素</text>
+ <text x="1370" y="262" font-size="24" fill="#FBBF24">← amber は文字色だけ</text>
```
