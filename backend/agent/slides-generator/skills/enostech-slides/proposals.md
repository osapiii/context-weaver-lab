## P-01 — neutral palette を zinc/stone 系へシフト (v9.30+)

### 背景

現在 `assets/themes.js` の `default.neutral` は Tailwind `neutral` 系
(完全な無彩色: FAFAFA / F5F5F5 / E5E5E5 / D4D4D4 / A3A3A3 / 737373 / 404040)
を採用している。一方 `highlight.base` は `1F2937` (実体は Tailwind `gray-800`)、
`canvas` は `FAFAF7` (warm off-white) と、
微妙に「無彩色 / クール / ウォーム」が混在している。

スライド全体を「インテリジェントで落ち着いた」印象に寄せる Genspark refine の文脈で、
neutral を **少し色温度を持った系** に統一することで、紙の温度感に近い読後感が出る。

### 候補比較

| パレット | 500 値 | キャラクタ | 紙の温度感 | 紫×アンバー (brand) との相性 |
|---|---|---|---|---|
| (現行) neutral | `737373` | 完全無彩色 | 中性 / 工業的 | ◎ (色相干渉なし) |
| zinc | `71717A` | わずかに青寄りクール | やや冷たい | ◯ (紫と同方向だが控えめ) |
| stone | `78716C` | わずかに茶/緑寄りウォーム | 暖かい | ◎ (アンバーと相性良) |
| slate | `64748B` | 青寄りクール (青みが強い) | 冷たい | △ (青みが brand 紫と干渉する場面あり) |
| gray | `6B7280` | わずかに青寄り (slate より控えめ) | やや冷たい | ◯ |

### 推奨: stone 系

理由:
- ENOSTECH の brand トーン (Vivid Purple #9212F3 + Light Magenta #E365FF +
  Amber アクセント) に **アンバー寄り** のニュートラルが合う
- 既存 `canvas: FAFAF7` (warm off-white) との連続性が高い
- 紙の温度感 (オフ印刷物の印象) に近づく
- 完全無彩色の `737373` は「データセンター感」が強く出てしまう場面がある

### 提案する具体値 (Tailwind stone)

```js
neutral: {
  50:  'FAFAF9',  // stone-50
  100: 'F5F5F4',  // stone-100
  200: 'E7E5E4',  // stone-200
  300: 'D6D3D1',  // stone-300
  400: 'A8A29E',  // stone-400
  500: '78716C',  // stone-500  (現行 737373 → 78716C)
  700: '44403C',  // stone-700  (現行 404040 → 44403C)
  900: '1C1917',  // stone-900  (現行 1F2937 = gray-800 から離れる)
}
```

ただし `gray900 = ink = 1F2937` は本文文字色として全テンプレで参照されているため、
`ink` を維持しつつ `neutral.900` だけを揃えるのは破綻リスクが高い。**選択肢**:

- (A) `ink` も `1C1917` (stone-900) に揃える → 全テンプレで文字色がわずかに変わる
- (B) `neutral.900` は `1F2937` のまま、500/700 だけ stone 系に揃える → 段階移行

### 副作用 / 移行リスク

1. **チャート系**: `_chart-style.js` の `axisColor: '6B7280'`、`valGridLine: C.gray200`
   が値で参照されている。`gray200` は token 経由で OK だが、`axisColor` はハードコード。
2. **コードブロック (CODE-1〜7)**: 1F2937 (Navy Dark) を背景に使っている。これは
   コード専用の固有色で **neutral とは独立**。影響なし。
3. **アイコン / 罫線**: `gray400` を罫線色として多用 (atoms-shape.js 等)。色温度が
   stone 系に変わると、紫 brand との相互作用で図解の印象が「やや暖かく」感じる。
4. **PNG エクスポート**: pptx 経由で PowerPoint 標準フォントレンダリングなので
   実害なし。
5. **既存デッキの再ビルド**: 同じ JSON で再ビルドしても色値が変わるだけ。レイアウト
   崩れはない。ただし「スクショベースの過去資料」と比較すると微差が見える。
