# ENOSTECH デザインシステム

> チームで資料デザインについて会話するための共通語彙集。
> **このドキュメントはデザイン管理用で、実装は宣言的に書いて OK**。
> 「タイルカード大の余白もうちょっと…」が通じる状態を目指します。

---

## 3 層構造

```
Atoms（デザイントークン）
  色・フォント・サイズ・余白・角丸などの原子的な値
  場所: assets/tokens.js
     ↓
表現パターン（Expression Patterns）
  1 つの視覚機能を持つ、再利用可能な要素
  例: 番号バッジ、チップ、タイルカード、フェーズブロック
     ↓
ページテンプレート
  表現パターンを組み合わせた、スライド単位の完成形
     ↓
デッキ
  複数ページテンプレートの連続。実際の資料
```

### 実装と管理は別である

- **実装時**: 現状通り `example-deck.js` を宣言的に書いて OK。Atomic Design 風の継承やフレームワーク化は不要
- **デザイン管理時**: 本書の表現パターン名で会話する
- **新規テンプレートを作る時**: 表現パターンの組み合わせで定義する

## A. Atoms — デザイントークン

詳細は `assets/tokens.js` と `references/_common/brand-tokens.md`。サマリのみ：

| カテゴリ | 主要トークン |
|---------|------------|
| 色（Primary）| `purple` `magenta` `ink` |
| 色（Accent）| `accent`（amber）`accentSoft` `accentDeep` |
| 色（Grayscale）| `gray50` 〜 `gray700`、`white`、`canvas`（#FAFAF7） |
| 色（補助）| `purpleSoft` `magentaSoft` |
| フォント | `Noto Sans JP` 統一 |
| サイズ | `titleXL: 32` `titleL: 20` `h2: 16` `body: 11` `caption: 9` 等 |
| レイアウト | `marginX: 0.40` `gutter: 0.22` `stripeW: 0.12` `cardRadius: 0.08` |
| 横断基準値 | `cardRadius: 0.08` `lineWidth: 0.25` `lineWidthStrong: 0.5` `cardPad: 0.30` `lineSpacingMultiple: 1.40` |

### 数値統一の根拠

**「角丸 0.08 / 罫線 0.25 / padding 0.30 / 行間 1.40」** は Genspark 系の知的UI寄り
の数値構成を ENOSTECH ブランドに引いてきた基準値。「主張は色で、区切りは余白で」を
実装可能な値に翻訳した結果。角丸を一段抑えて知的、罫線を薄くして上品、padding と
行間を広げて読みやすく、というトーンを 4 軸で統一する。詳しくは
`brand-tokens.md` § 4。

## B. 表現パターン カタログ

用途別に整理。各項目は「和名（英名）」「用途」「バリエーション」「使用テンプレート」。

### B-1. 識別要素 — 並列性・順序を示す

#### 番号バッジ（numBadge）
円形の数字マーカー。項目の並列性と順序を視覚化。

| バリエーション | 外観 | 使用例 |
|-------------|------|-------|
| 塗り大 (0.48") | `purple` or `accent` + 白文字 | 17 Tile 2×2 の主要タイル |
| 薄塗り大 (0.48") | `purpleSoft` or `accentSoft` + 濃色文字 | 17 Tile 2×2 の副タイル |
| 塗り中 (0.40") | `purple` or `accent` + 白文字 | 15 Profile 強みカード |
| 枠線大 | purple outline（塗りなし） | 04 Content 要点カード |

#### コーナー番号（cornerNum）
タイル右上に配置する、塗りなしの小テキスト番号。タイル本体の情報を邪魔しない控えめな識別子。

- 非アクティブ: `gray400` / `gray300`
- アクティブ時: `purple` / `accentDeep`

#### 大数値（statNum）
存在感を出したい大きな数字。
- セクション番号 (120pt, purple) — 03 Section Divider
- 統計値 (22pt, white bold on black pill) — 14 総予算

---

### B-2. ラベル要素 — 属性・状態・位置を示す

#### チップ（chip）
小さな角丸矩形（rectRadius 0.13〜）。位置・状態・属性を短く示す。

| バリエーション | 外観 | 使用例 |
|-------------|------|-------|
| アクティブ | `purple` 塗り + 白 bold | ナビ現在位置、08 featured chip |
| 非アクティブ | `gray100` + `gray500` | ナビ他、08 default chip |
| 薄紫 | `purpleSoft` + `purpleDeep` | 08 category chip |

#### ピル（pill）
大きめの角丸矩形。値や注釈・状態を囲む。

| バリエーション | 外観 | 使用例 |
|-------------|------|-------|
| 黒ピル | `ink` + 白 bold | 11 フェーズ名、12 下段ピル |
| 紫ピル | `purple` + 白 bold | 11 時期ピル、12 月額稼働費ヘッダー |
| アンバーピル | `accent` + 白 bold | 12 マイルストーン、達成報酬 |
| 複合ピル | 黒地 + 白小ピル + 大数値 | 14 総予算ピル |

#### 強調タグ（tag） 超小型のピル。**セル内や要素近接の emphasis** に使う（チップより小さく、主張度が上がる）。ヘルパー `addTag(s, text, x, y, variant)` で配置。

| バリエーション | 色 | 使いどころ |
|-------------|---|---------|
| `important` | 🔴 赤 (`#DC2626`) + 白 bold | 最優先事項、赤マーカー的な扱い |
| `brand` | テーマの `brand` 色 + `brandContrast` | 新機能、ブランド誘導（テーマ連動） |
| `accent` | テーマの `accent` 色 + `accentContrast` | 注意喚起、ポジティブ強調（テーマ連動） |
| `info` | `gray200` + `gray700` | 補助情報、ラベル |

**特徴**:
- サイズは `sm` (h=0.22, 8.5pt) / `md` (h=0.28, 10pt default) の 2 段階
- 完全な角丸ピル（rectRadius = h/2）
- 狭いセル・右上・行末などの **狭いスペースに置ける**

**使用例**:
```js
addTag(s, '重要', 9.05, 2.30, 'important', { size: 'sm' });
addTag(s, 'New', 4.50, 1.80, 'brand');
addTag(s, '注意', 6.20, 3.10, 'accent');
```

#### 黒ラベル（blackLabel）
小さな角/角なしの黒背景ラベル。ピルではなく矩形形式。
- 縦長 (2 行) — 16 "解消手段"
- 横長 — 15 "プロジェクトでの強み"、10 KV 行ラベル

### B-3. 線・棒 — 区切り・ガイド

#### アクセント棒（accentBar）
左端に沿った細い縦線。視線のガイド。
- 外側アクセント (0.04" purple) — 04 コンテンツ全体の左端
- カード内部アクセント (0.05" purple) — 04 要点カード左端

#### 左端ストライプ（leftStripe）
全スライド共通の左側色帯。「ここは ENOSTECH の資料」を示す帯。
- 0.12" × 全高 / `purpleSoft`
- 黒背景スライド（03）のみ省略

#### 水平区切り線（divider）
要素を区切る横線。

| バリエーション | 使用例 |
|-------------|-------|
| 細 (0.5pt `gray200`) | 02 目次項目間、08 Feature 下、15 ロゴ帯上 |
| 太 (1.5pt `ink`) | 10 テーブル上下 |

### B-4. 強調・誘導

#### 矢印（arrow）
テキスト記号として配置する矢印。

| バリエーション | 使用例 |
|-------------|-------|
| → (32pt accent) | 05 これまで→これから の間 |

#### ブレット（bullet）

### B-5. カード類

カードは「矩形 + 枠線 + 内容ブロック」の基本形。用途・サイズ・内容で細分化。

#### タイトルブロック（titleBlock） ・最上位
全スライド共通で使う、タイトル + サブコピーを統合した引用カード風ブロック。
- 背景: `purpleSoft` (薄紫)
- 左端縦棒: `purple` 幅 0.06"
- 内部: タイトル (20pt bold ink) / サブコピー (11pt gray700)
- オプション: 上部に黒ラベル（eyebrow）を追加可能
- サイズ: 幅 9.2" × 高さ 1.05"
- 配置: ナビあり y=0.55、ナビなし y=0.30
- ヘルパー: `addTitleBlock(s, title, sub, opts?)`

用途: **本編スライド全てのタイトル領域**。表紙・閉じ・セクション扉・ビジュアル主体を除く全スライド。

#### タイルカード小（tileSmall）
3×3 グリッド用、最コンパクト。
- 丸角 0.08"、枠線 `gray200`
- 内容: コーナー番号 / タイトル / 1-2 行本文
- dim バリアント (`gray50` 塗り、「準備中」等)

#### タイルカード中（tileMedium）
3×2 グリッド用、中密度。
- 丸角 0.10"、枠線 `gray200`
- 内容: コーナー番号 / 小ラベル / タイトル / 2-3 行本文
- highlight バリアント (`purpleSoft` or `accentSoft` 塗り、色によって意味付け可)

#### タイルカード大（tileLarge）
2×2 グリッド用、ゆったり情報。
- 丸角 0.12"、枠線 `gray200`
- 内容: 番号バッジ大 / タイトル h2 / 3-4 行本文
- バッジ色のバリエーション (purple塗り/amber塗り/薄塗り) で強弱を作れる

#### プロダクトカード（productCard）
商品カタログ用（07）専用の縦長カード。
- 番号 / 商品名 (15pt bold) / タグ (10pt purple) / 説明 (10pt) / 区切り / カテゴリ (9pt)
- featured バリアント (`purpleSoft` 背景)

#### 引用カード（quoteCard）
「このページの要点」型の強調ブロック。
- `purpleSoft` 背景 / 左端アクセント棒 / 小ラベル / 太字本文

#### 比較カラム（compareColumn）
「これまで」「これから」の 2 カラム比較。
- 丸角 0.12"、背の高いカード
- これまで: `gray100` + `gray500` テキスト（過去感）
- これから: `accentSoft` + `accentDeep`（未来感、温かさ）

#### 3 カラムカード（triColumnCard）
並列 3 概念の縦長カード。
- 上辺ストライプ (purple / accent / ink の 3 色) / 大番号 (gray300) / タイトル / 本文 / 区切り / 要点 3 つ

### B-6. 複合ユニット — 特定用途の組み合わせ

#### フェーズブロック（phaseBlock）
プロジェクト 1 フェーズの情報セット。
- 黒ピル (フェーズ名) / タイトル / 本文 / アウトプット箱 / 時期ピル

#### ガントセル（ganttCell）
スケジュール表 1 行の帯 + マイルストーン + フェーズピル。
- 行ラベル (黒ラベル) / バー (`purpleSoft`) / マイルストーンピル (amber) / フェーズピル (ink)

#### KV 行（kvRow）
項目名 / 値を整然と並べる行。
- 左: ラベル (`gray50` 背景 + bold) / 右: 値 (rich text 対応)
- 上下に divider (太) で囲むのが定番

#### プロフィールヘッダー（profileHeader）
自己紹介スライドの冒頭固定セット。
- 上: リードバナー (purpleSoft + 左アクセント棒) / 下: 写真 + 名前 + 強みカード群

#### 強みカード（strengthCard）
横長カード + 左に番号バッジ中 + 右にタイトル/本文。
- タイルカード大の「横並び版」
- バッジ色は混在 OK (purple / amber 混ぜて単調さ回避)

#### ロゴ帯（logoStrip）
下部の取引先ロゴ並び。
- 上部細 divider / 左端にラベル (9pt gray500) / 右に N 個のロゴスロット

#### 吹き出し（calloutBox）
引用・抜粋の強調。
- `gray100` 塗り矩形 + 左三角（三角は省略可）/ rich text 本文（キーワードを accent/purple で強調）

#### 結論バナー（conclusionBanner）
スライド下部の「だからこうすべき」帯。
- `accentSoft` 塗り + 黒ラベル (縦長) + 本文（キーワードを `accentDeep` で強調）

#### 統計ブロック（statBlock）
大きな数字を目立たせる専用セット。
- 総額ピル (複合ピル) / 内訳テーブル / 注釈

#### ヒーロービジュアル（heroVisual）
スライド中央の大きな図・画像領域。
- 丸角 0.14" / 枠線 gray200 / 影 (blur 8, opacity 0.06) / 内部にモック or 実画像

### B-7. ビジュアル要素

画像・アイコン・プレースホルダー系。

#### 画像プレースホルダー（imagePlaceholder）
実画像を入れるまでの仮置き。寸法を明示するのが特徴。
- グレー背景 (`gray100`) + 枠線 (`gray300 0.75pt`) + 丸角 0.08
- 中央に ▢ シンボル (32pt gray400) + 寸法テキスト (11pt bold gray500, `WWW × HHH` px 換算)
- オプション: 左上に小ラベル (9pt, 'プロダクト画面' 等)
- ヘルパー: `addImagePlaceholder(s, x, y, w, h, { label, radius })`
- 実運用時は `s.addImage({ path, x, y, w, h, sizing: { type: 'cover', w, h } })` に差し替え

#### 実画像領域（imageFrame）
プレースホルダーから本番画像に置き換わった状態。
- プレースホルダーと同じ座標・サイズで `addImage`
- `sizing: { type: 'cover', w, h }` で枠内フィット（アスペクト比が違う画像でも綺麗に）
- または `sizing: { type: 'contain', w, h }` で全体表示（余白が出る）

#### アイコン円（iconCircle）
丸背景 + アイコン文字（絵文字 or 小図）。
- 直径 1.10" の OVAL / `brandSoft` 背景 / 中央に 38pt `brand` 色のシンボル
- シンボル例: ⚡ スピード, ◎ クオリティ, ⚑ スケーラビリティ
- 下部にタイトル (17pt bold) + 装飾アンダーライン (LINE 2pt accent 色 0.60" 幅)

#### キャプション付き画像（captionedImage）
画像 + その下に短いキャプション。
- 画像プレースホルダー下部に 11pt gray500 のキャプションテキスト

### B-8. 参照・引用要素

出典・注釈・引用を扱う要素。ファクトベース資料で頻出。
**URL は表示せず、タイトルを青文字（`#0563C1`）ハイパーリンクで表示するのが標準**。
**本文内に `(1)` `(12)` のインライン参照番号を埋める「3 段構え」が推奨**。

#### インライン参照番号（inlineRef）
本文テキスト内に `(1)` `(12)` のような **小さめの青文字ハイパーリンク** を埋める要素。
主張の直後に配置し、クリックで出典 URL に直接飛ばす。

- 色 `#0563C1` + underline / 周囲より 1pt 小さめ（本文 13pt に対して 10pt 等）
- 番号は末尾の**参考情報集ページ (SR) の行番号**と一致させる
- `slide.addText([...runs])` の runs 配列に `inlineRef(num, url)` を混ぜて使う
- ヘルパー: `inlineRef(num, url, opts?)` → text run を返す関数
  - `opts.tooltip`: ツールチップ文字列（省略時 `出典 (N)`）
  - `opts.size`: フォントサイズ（省略時 10pt）

**使い分けの指針**:
- **インライン番号**は「どの主張がどの出典に基づくか」を読者がその場で確認したい時（UX 向上）
- **脚注**はスライド単位で主な出典を 1〜2 個示したい時
- **参考情報集ページ**は全出典のマスターリスト（必須）

#### 脚注（footnote）
スライド下部、フッター直上に `※ 出典: ...` を並べる小さな注釈。
- フォントサイズ 8.5〜9.5pt / プレフィックス `※` / 色 `gray500`
- **`{label, url}` 形式で渡された時は URL を表示せず、label を青リンク化**（`#0563C1` + underline）
- 複数の出典を扱う場合は、下から順に積み上げて表示
- ヘルパー: `addFootnote(slide, items, opts)`
  - 単一文字列: `addFootnote(s, '本調査は n=713 の回答に基づく')`
  - 出典リスト: `addFootnote(s, [{label: 'IPA DX動向', url: 'https://...'}, ...])` → URL 非表示、label のみ青リンク

#### 参考情報集テーブル（referenceTable）
デッキ末尾に入れる標準 3 列テーブル。**URL を持つ行はタイトル列が青リンク化**（URL 文字列は一切表示しない）。
- ヘルパー: `addReferenceTable(slide, rows, opts?)`
- 行の形: `{ category, title, url?, source }`
- URL があれば `title` が `#0563C1` + underline のハイパーリンクになる
- URL がなければ通常の黒文字

**全デッキで必須の構成要素**。デッキ末尾に DATA-4 パターンで 1 枚配置する。

#### 引用タイプの使い分け

| 方式 | 見た目 | 使い所 |
|-----|-------|-------|
| **インライン番号** | 主張の直後に `(1)` 青リンク | 主張と出典を 1 対 1 で対応づけたい時（UX 良） |
| 脚注スタイル | `※ 出典: {青リンクタイトル}` をスライド下部 | 主張に対する出典の明示（推奨） |
| 参考情報集ページ | 3 列テーブル（カテゴリ / 青リンクタイトル / 出典年） | デッキ末尾で全ソースを集約（必須） |
| キャプション | 図表直下に `出典: ...` | グラフ・表の出典 |

**ENOSTECH デフォルトは「インライン番号 + 脚注 + 末尾の参考情報集ページ」の 3 段構え**。
ファクト密度の低いスライドではインライン番号を省いて脚注だけにしてよい。末尾の参考情報集ページは**常に必須**。

#### 多行データテーブル（longTable）
行数が多い汎用 3 列テーブル。`addTable` の拡張運用。
- 1 列目: カテゴリラベル (`gray50` 背景 + bold)
- 2 列目: メインテキスト（タイトル）
- 3 列目: 補足情報（出典・日付・著者）
- ヘッダー行: `ink` 背景 + 白 bold
- 行高さ: 0.40 前後（7-8 行で収まる設計）

#### 比較表（comparisonMatrix）
評価項目 × 対象の星取り表。
- ヘッダ 1 列目: 評価項目（黒背景 + 白文字）
- ヘッダ 2 列目以降: 対象（**推しは brand 色背景**、他はグレー）
- データセル: `◯` (brand), `△` (accent), `×` (gray400) の 3 段階記号
- 下に凡例（3 項目横並び）+ 脚注
- ヘルパー関数 `cell(rating, note)` で評価+コメントを統一形式に

#### トレードオフスライダー（tradeoffSlider）
4 軸の優先順位を横長スライダーで視覚化。
- 各行 = 1 軸（例: 納期/品質/スコープ/予算）
- 左側: 軸名を黒ラベルに（ROUNDED_RECTANGLE, ink 塗り + 白 bold 13pt）
- 中央: 横長スライダー（gray200 土台 3pt + 4 つのドット）
- アクティブドット（優先順位の現在位置）: brand 色の大きな丸（0.32"）+ 白抜き順位番号
- 非アクティブドット: gray300 の小さな丸（0.18"）
- 右側: 優先順位の理由テキスト（11pt gray700、1-2 行）
- ヘッダー: 「優先度 高（固定）」「優先度 低（調整可）」を両端に配置
- フッター: accentSoft 背景の判断原則バナー + 注意事項の脚注

### G. クローム — 画面の骨格

#### ページナビ（pageNav）
上部のセクションチップ列。本編スライドで標準装備。
- `DEMO_SECTIONS` 配列 + currentIdx で描画
- 現在: `purple` 塗り + 白 bold / 他: `gray100` + `gray500`
- ヘルパー: `addChromeNav(slide, sections, idx)`

#### 左下ロゴ（chromeFooterLogo）
フッター帯に公式ロゴ（horizontal-black）を配置。ブランディングはテキスト繰り返しではなくロゴに委ねる。
- 位置: `x: marginX, y: footerY - 0.02` / サイズ: `w: 1.15, h: 0.24`（4.8:1 アスペクト維持）
- ヘルパー: `addChromeFooterLogo(slide)`

#### ページ番号（chromePage）
右下のページ番号。
- "04 / 21" 形式 / 現在値 bold `gray500`、全体 `gray400`
- ヘルパー: `addChromePage(slide, current)`

#### 上辺／下辺 2 色バー（dualBar）
表紙・閉じのみに使う、purple + accent の 2 色水平バー。
- 上辺 (0.04") / 下辺 (0.045")
- purple 65% + accent 35% で分割

### H. 図解 — 構造を視覚化する

情報間の「関係性」「階層」「流れ」を形で表す要素。一部はページテンプレートとしても採用。

#### フロー図（flowDiagram）
横一列のステップ + 矢印で、プロセスの流れを示す。
- ステップ数: 3〜5 推奨
- 各ステップは丸角矩形（強調の 1 ステップのみ塗り、他は枠線 or 薄塗り）
- 矢印 (→, 32pt accent) で次へ誘導
- 各ステップに番号 + タイトル + 短い説明

用途: 業務フロー、導入ステップ、ユーザージャーニー、意思決定プロセス

#### サイクル図（cycleDiagram）
循環プロセスを表す、4 つのノード + 循環矢印。
- 4 ノード配置（上・右・下・左）、または 3 ノードの三角配置
- 各ノードは円形（バッジ大）+ タイトル + 短説明
- 矢印は ↓ → ↑ ← を配置して循環感を演出
- 中央にサイクル全体の名前

用途: PDCA、反復プロセス、継続改善、学習サイクル

#### ピラミッド図（pyramidDiagram）
上→下に広がる階層構造。優先度・MECE の可視化。
- 3〜4 層、各層は台形 or 矩形（下ほど広く）
- 上層: 最も重要（`purple` 塗り）
- 中層: 補助（`purpleSoft`）
- 下層: 土台（`accentSoft` or `gray100`）
- 右側に層ごとのラベル + 説明文

用途: 優先度、価値階層、戦略ピラミッド、マズロー型概念図

#### マトリクス 2×2（matrix2x2）
2 軸 × 4 象限の分類図。ポジショニング・優先度分析・競合比較。
- 縦横の 2 軸、端にラベル（高/低 or 具体名）
- 4 象限それぞれに分類名 + 要素リスト
- 1 象限を強調色（`purple`）で「ここが狙い所」を示せる
- 色分けの定型: 右上 = `purple`(主役), 右下 = `accent`(次候補), 左上/左下 = `gray100`/`gray50`

用途: 優先度分析、競合マッピング、ポートフォリオ分析、SWOT の簡易版

#### ベン図（vennDiagram）
2〜3 円の重なりで共通点・交差を示す。
- 円は半透明色（`purpleSoft` と `accentSoft` をオーバーラップ）
- 重なり部分に「両方の特徴を備える」ラベル
- 2 円 or 3 円

用途: 差別化ポジション、兼ね備える特性、市場の共通領域

#### ツリー図（treeDiagram）
親→子の階層分岐。組織図・ロジックツリー・MECE 分解。
- 上から下への分岐、または左から右への分岐
- ノードは矩形（ルート: `purple` 塗り、子: gray200 枠線 + 白塗り）
- 接続線: LINE shape、`gray300`

用途: 組織図、ロジックツリー、課題分解、システム構成

#### 放射図／ハブ&スポーク（hubSpokeDiagram）
中央のコアと周辺ノードの関係図。
- 中心: 大きな円（`purple` 塗り）+ コア概念
- 周辺: 4〜8 個のノード（小円 or 矩形）
- 放射状の接続線（LINE shape）
- 周辺ノードの色は役割で使い分け

用途: コアサービスの派生、中心戦略の枝分かれ、エコシステム図

## C. 新しいページを組み立てる時のガイド

### Step 1: 既存テンプレートで足りるか確認
既存のどれかで実現できるなら、それを使う。`slide-patterns.md` の判断フロー参照。

### Step 2: 足りなければ、表現パターンを組み合わせる
例: 「タイルカード大を 4 列並べたい」→ 17 の colW を調整 or 新規組み立て。
例: 「KV 行の左ラベルを黒ピルにしたい」→ 10 の blackLabel をピル化して派生。

### Step 3: 表現パターンにも無いなら、新規カタログ化を検討
同じ要素が 3 回以上出てくる見込みなら、新しい表現パターンとして本書に追加する価値あり。1 回きりなら atoms (tokens) レベルで書き切って OK。

## D. 命名のルール

チームで意思疎通しやすくするため、以下の命名を守る。

### 和名
- 機能が明確な短い名前（1〜4 語）
- 例: 番号バッジ / 引用カード / 結論バナー
- 「〜カード」「〜バナー」「〜ピル」などの型名で終わる
- 同じ機能の細分バリアントは「大・中・小」または形容詞で（例: タイルカード大）

### 英名（コード識別子）
- camelCase
- 機能を示す動詞 or 名詞（`numBadge`, `chipActive`, `phaseBlock`）
- Claude / エンジニアが書くコード内の関数・変数名に使う

### バリエーションの識別
- `タイルカード大` = 2×2 用
- `タイルカード中` = 3×2 用
- `タイルカード小` = 3×3 用
- 同じ表現パターンが色違いを持つ場合は「紫ピル / アンバーピル / 黒ピル」のように色を付ける

## E. 変更管理

表現パターンを変更する時のフロー：

1. **まず本書を更新**（設計の変更を言語化する）
2. 変更対象の表現パターンがどのテンプレートで使われているか、対応する例を確認
3. 影響範囲を評価したうえで、実装 (`example-deck.js`) を変更
4. 生成 PPTX を QA して regression を確認
5. （必要なら）HTML リファレンス (`components/`) も合わせる

**このフローを守ることで、「コードだけ直したら他のスライドが壊れた」を防げる**。
