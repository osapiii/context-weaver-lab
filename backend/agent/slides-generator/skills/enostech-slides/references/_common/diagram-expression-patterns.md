# ダイアグラム表現パターン カタログ

> ダイアグラムを支える **原子要素（atomic pieces）** の仕様書。
> すべての実装ヘルパーは `scripts/diagram-patterns.js` の「ダイアグラム原子要素」セクションに定義されている。
>
> **このドキュメントを読む前提**: `references/_common/design-system.md` の 3 層構造（Atoms → 表現パターン → ページテンプレート → デッキ）を理解していること。本書はこの 3 層構造を **ダイアグラム専用に拡張** したもの。

---

## 全体像

```
Atoms（トークン）
  diagramPalette: 6 トラック × 3 階調（deep/mid/bg）
  diagramSize:    ドット半径・軸太さ等の固定値
     ↓
表現パターン（本書で扱う 10 原子要素）
  データドット、4象限背景、十字軸、バーカード、上バーカード、
  番号バッジ、ハブノード、放射コネクタ、タイムラインドット、
  インサイトコールアウト
     ↓
ページテンプレート（DIAG-XX）
  サイクル、階段、比較、ピラミッド、
  タイムライン、放射、2×2 マトリクス、2軸プロット
```

---

## Atoms（トークン）— `assets/tokens.js`

### diagramPalette

テーマ横断で固定される 6 系統のカラーパレット。

| トラック | deep | mid | bg | 用途例 |
|---------|------|-----|-----|-------|
| `navy` | `#1F3A5C` | `#4A6A8E` | `#EDF1F7` | 基幹・信頼性 |
| `teal` | `#2A6B6B` | `#5B9A9A` | `#E6F0F0` | 技術・分析 |
| `purp` | `#6B3AA8` | `#9B6FC7` | `#F2ECFA` | ブランド・強調 |
| `gold` | `#A67A2E` | `#C9994D` | `#F7EEDB` | 達成・成果 |
| `sage` | `#4F7D5E` | `#7DA087` | `#EAF1EC` | 成長・安定 |
| `rose` | `#A34A5E` | `#C27B8B` | `#F5E6E9` | 警告・注意（柔） |

**役割分担（階調）**:
- `deep` → 濃色: 文字、アイコン、主張部分、アクセントバー
- `mid`  → 中色: 塗り本体（ピラミッド層、ヘッダー、バーカード）
- `bg`   → 淡色: 背景、ハロー、タイントセル、淡色カード

### diagramSize

ダイアグラム要素のサイズ定数。

| キー | 値 | 用途 |
|------|----|----|
| `dotR` | 0.15 | 標準データドットの半径 |
| `dotRHi` | 0.22 | ハイライトドットの半径 |
| `dotHaloOut` | 0.14 | ハロー外側の膨らみ |
| `dotHaloIn` | 0.07 | ハロー内側の膨らみ |
| `axisWidth` | 1.5 | 十字軸の太さ |
| `axisColor` | `6B7280` | 軸の色（gray500 固定） |
| `connectorWidth` | 1.3 | 放射コネクタの太さ |
| `stepBadgeR` | 0.21 | 番号バッジの半径 |
| `cardRadius` | 0.10 | カードの角丸 |
| `cardBarW` | 0.06 | カード左色バーの幅 |
| `cardTopBarH` | 0.08 | カード上色バーの高さ |
| `hubR` | 0.45 | ハブノード（放射図中心）の半径 |
| `spokeR` | 1.75 | 周辺ノード配置半径 |

## 表現パターン（10 要素）

各要素は `scripts/diagram-patterns.js` にヘルパー関数として実装済み。テンプレート側からは **宣言的に呼ぶだけ** でよい。

---

### 1. データドット（dataDot）

プロット図・タイムライン・放射図で使う円形マーカー。

**バリエーション**:
| タイプ | 外観 | 使用例 |
|-------|------|-------|
| 標準 | 単色塗り + 白枠 | プロット図の競合 |
| ハイライト | 2 層ハロー付き、大きめ | プロット図の自社、注目点 |

**実装**:
```javascript
addDataDot(s, { cx, cy, pal, code, highlight: false });
addDataDot(s, { cx, cy, pal: D.purp, code: '自社', highlight: true });
```

**使用**: 09 2軸プロット

---

### 2. 4 象限背景（quadrantBg）

2 軸の 4 象限を色分けして表示する淡色矩形。軽い枠線つき。

**バリエーション**:
- 4 象限すべてを異なる色で塗る（プロット図）
- 1 象限だけ色を入れて他を `gray100`（ゾーン強調）

**実装**:
```javascript
addQuadrantBg(s, {
  x, y, w, h,
  bg: D.purp.bg,
  label: '★ スター領域',
  labelColor: D.purp.deep,
  align: 'right',   // 'left' or 'right'
});
```

**使用**: 09 2軸プロット、08 2×2 マトリクス（tinted セル）

---

### 3. 十字軸（axisCross）

プロット図中央の十字線。グレー単色 1.5pt。

**実装**:
```javascript
addAxisCross(s, { px, py, pw, ph });
```

**使用**: 09 2軸プロット

---

### 4. バーカード（barCard）— 左色バー付きカード

左端に色バーを持つカード。ラベル + サブの 2 段構成。

**バリエーション**:
| タイプ | 外観 | 使用例 |
|-------|------|-------|
| 標準 | 左バー + ラベル + サブ | タイムラインのマイルストーン |
| キー文字付き | 左バー + 大文字キー + ラベル + サブ | PDCA サイクル |

**実装**:
```javascript
// 標準
addBarCard(s, { x, y, w, h, pal, label, sub });

// キー文字付き（PDCA 等）
addBarCard(s, { x, y, w, h, pal: D.navy, keyChar: 'P', label: '計画', sub: '目標と手段を設計' });
```

**使用**: 02 サイクル、06 タイムライン

---

### 5. 上バーカード（topBarCard）

上端に色バーを持つ大きめのカード。中央揃えでタイトル + サブ。

**バリエーション**:
| タイプ | 外観 | 使用例 |
|-------|------|-------|
| 白背景（デフォルト） | 白 + 色バー + 色枠 | マトリクスの副セル |
| 淡色背景（tinted） | `bg` 色 + 色バー + 色枠 | マトリクスの主セル（重点象限） |

**実装**:
```javascript
addTopBarCard(s, { x, y, w, h, pal, label, sub, tinted: false });
addTopBarCard(s, { x, y, w, h, pal: D.sage, label: '重点施策', sub: '効果高×実行しやすい', tinted: true });
```

**使用**: 08 2×2 マトリクス

---

### 6. 番号バッジ（stepBadge）

円形の番号マーカー。白枠で縁取られているため、背景の境界線をまたいで置ける。

**実装**:
```javascript
addStepBadge(s, { cx, cy, pal, num });
```

**使用**: 03 ステップアップ（階段の上端に跨る配置）

---

### 7. ハブノード（hubNode）

放射図の中央に置く円形ノード。ハロー + 塗り + 2 段テキスト。

**実装**:
```javascript
addHubNode(s, {
  cx, cy, r,
  label: 'FactHub',
  sub: 'AI データ基盤',
  // デフォルトはテーマの brand 色
});
```

**使用**: 07 放射図

---

### 8. 放射コネクタ（spokeConnector）

中心ハブから周辺ノードへ伸びる単色の線。ノード端の手前で止まる計算つき。

**実装**:
```javascript
spokes.forEach((sp, i) => {
  const angle = (i / spokes.length) * 2 * Math.PI - Math.PI / 2;
  addSpokeConnector(s, { cx, cy, hubR, nodeR: spokeR, angle, palMid: sp.pal.mid });
});
```

**使用**: 07 放射図

---

### 9. タイムラインドット（timelineDot）

タイムライン基線上の位置マーカー。白リング + 色塗り中心の 2 層構造。

**実装**:
```javascript
addTimelineDot(s, { cx, cy, pal });
```

**使用**: 06 タイムライン

---

### 10. インサイトコールアウト（insightCallout）

主張を示す右側パネル。上色バー + タイトル + 本文 + オプション注記。
**マッキンゼー品質の肝**: プロット図や比較図で「絵だけで終わらせない」ためのキーメッセージ格納器。

**実装**:
```javascript
addInsightCallout(s, {
  x, y, w, h,
  title: 'INSIGHT',
  body:  'FactHubは高差別化×中〜高汎用性のスター領域で独自の優位を確立',
  note:  '競合はコモディティ〜価格競争領域に集中。差別化領域の空白を捉えて先行。',
  palColor: D.purp.deep,
  palMid:   D.purp.mid,
});
```

**使用**: 09 2軸プロット（他パターンでも流用可）

## 実装確認

全 10 要素のヘルパー関数は `scripts/diagram-patterns.js` の以下のコメントブロックに定義されている：

```
// ═════════════════════════════════
// ダイアグラム原子要素（Expression Patterns）
// ═════════════════════════════════
```

新しいダイアグラムを追加したくなった場合、まずここに原子要素として切り出せる部分がないかを検討し、**あればヘルパー関数化してから** テンプレートを書くこと。これにより、
- 他のダイアグラムでも再利用できる
- サイズ・色の一括変更が効く
- テンプレートコードが宣言的になって読みやすい

という 3 層アーキテクチャの恩恵が得られる。
