# Chart パターン (CHART-01〜09)

## 役割

スライド内に「定量データの可視化」を入れたいときに、`scripts/render/charts/*.js`
の純粋関数として 9 種類の "**部品**" を用意してある。スライドテンプレ自体は
情報レイアウト軸 (CHART-A1〜A4) で 4 種類だけ。両者を JSON で組み合わせる。

## チャート種別 (9 種)

| ID | 名前 | 一言で | データ JSON の核 |
|----|------|-------|----------------|
| CHART-01 | 縦棒 (BAR-COL) | 期間/項目の量比較。最頻使用 | `categories[]` + `series[{name, values[]}]` |
| CHART-02 | 積み上げ縦棒 (BAR-STACKED) | 構成比の推移 | 同上 |
| CHART-03 | 横棒 (BAR-HORIZ) | 長いラベル、ランキング | 同上 |
| CHART-04 | 折れ線 (LINE) | 時系列推移 | 同上 |
| CHART-05 | 棒+線複合 (COMBO) | 売上額(棒)+成長率(線) | `bar_series[]` + `line_series[]` |
| CHART-06 | ウォーターフォール | 起点→増減→着地 | `items[{type:"start"|"plus"|"minus"|"end", value}]` |
| CHART-07 | ドーナツ (DOUGHNUT) | 構成比 (3-5 セグメント) | `items[{name, value}]` |
| CHART-08 | 散布図 (SCATTER) | 2 変量の関係、ポジショニング | `series[{name, points[{x,y}]}]` |
| CHART-09 | レーダー (RADAR) | 多軸スコア比較 | `axes[]` + `series[{name, values[]}]` |

## スライドテンプレ (4 種)

| テンプレ | レイアウト | 使う場面 |
|----------|----------|---------|
| CHART-A1 | チャート単体 (大きく 1 枚) | 1 つの強いメッセージを伝える |
| CHART-A2 | 左チャート + 右テキスト 3 観点 | 数字 + 解釈をセットで提示 |
| CHART-A3 | 上チャート + 下 3 カラム (Why/So What/Now What) | コンサル定番、論理を 3 角度から |
| CHART-A4 | 上下 (or 左右) 2 チャート並列 | 推移と構成、自社と競合 などの対比 |

## 組み合わせ JSON の書き方 (CHART-A1 の例)

```json
{
  "id": "S07",
  "template_id": "CHART-A1",
  "title": "売上は 4 年で 1.75 倍",
  "subtitle": "粗利率も後半 2 年で改善傾向。",
  "section_id": "ch01",
  "chart": {
    "template_id": "CHART-01",
    "categories": ["2022", "2023", "2024", "2025"],
    "series": [
      { "name": "売上 (億円)", "values": [120, 145, 180, 210] },
      { "name": "粗利 (億円)", "values": [40, 55, 72, 88] }
    ],
    "show_value": true
  },
  "chart_caption": "出典: 自社決算資料"
}
```

CHART-A4 だけ `chart_top` / `chart_bottom` の 2 つを持つ。

## 選び方のクイックガイド

```
「データから 1 つの主張を出したい」
  → CHART-A1 (チャート単体)
「グラフ + そこから読み取れる 3 観点を一緒に伝えたい」
  → CHART-A2 (左チャート + 右テキスト)
「Why / So What / Now What で議論を構造化」
  → CHART-A3 (上チャート + 下 3 カラム)
「2 つの角度から見せたい (推移と構成、自社と競合)」
  → CHART-A4 (2 チャート並列)
```

チャート種別の選び方:

```
推移 / 比較 / ランキング
  - 期間で量の変化       → CHART-01 (縦棒)
  - 期間で構成比の変化   → CHART-02 (積み上げ縦棒)
  - 5+ 項目の長いラベル   → CHART-03 (横棒)
  - 連続的なトレンド     → CHART-04 (折れ線)
  - 額と率を同時に       → CHART-05 (棒+線複合)
  - 起点 → 増減 → 着地    → CHART-06 (ウォーターフォール)
構成比
  - 1 時点の構成比 (3-5 個) → CHART-07 (ドーナツ)
関係
  - 2 変量の関係・配置    → CHART-08 (散布図)
  - 5-8 軸の多角評価     → CHART-09 (レーダー)
```

## 設計上の絶対ルール

### G-CHART-1 「文字 + 数字 + チャート」を 1 セットで

チャートだけだと「数値の生表」と変わらない。**サブコピー (アクションタイトル)
で「何を読むべきか」を明示** する。

例:
- ❌ 「年次売上推移」(トピック)
- ✅ 「売上は 4 年で 1.75 倍。粗利率も後半 2 年で改善傾向」(アクション + 数値)

### G-CHART-2 系列数は最大 4 まで

5 系列以上はパレットに頼った識別が破綻し、認知負荷も急増する。

### G-CHART-3 mono テーマで黒の重ね打ちを避ける

`_chart-style.js` の `chartPalette` は brand → accent → brandSoft の順に
返す。複数系列のときに同色 (黒) が連発しないよう、テンプレ側で色を
ハードコードしないこと。色は必ずパレット経由 (CHART-XX の標準実装に従う)。

### G-CHART-4 出典は必須

`chart_caption` で「出典: ...」を必ず添える。コンサル流儀ではすべての
データスライドにソースラインがある。

### G-CHART-5 `addChart` の opts は使い回さない

pptxgenjs はオプションオブジェクトを内部でインプレース変更する。
複数チャートを描画するときに同じ opts を 2 回渡すと壊れる。
**`_chart-style.js` のファクトリ関数経由で毎回新規生成** する設計に
なっているので、これを破らないこと。

## ハマりポイント

1. **ウォーターフォールの透明ベース棒** — 背景色 (canvas = #FAFAF7) で
   塗ることで背景に溶け込ませる。白背景でないキャンバスでは「白い四角」が
   見えてしまう。
2. **データラベルのシリーズ別制御** — pptxgenjs の `showValue` は系列毎の
   制御ができない。必要なら shape ベース (slide.addText) で別途描画する。
   CHART-06 ウォーターフォールがこのパターン。
3. **フォントは Noto Sans JP を毎回指定** — 軸/凡例/データラベル全て
   `fontFace: F.jp` を渡す (`_chart-style.js` で集約済み)。
4. **ドーナツは CHART-A4 と相性が悪い** — CHART-A4 の上下半分領域だと
   ドーナツが極小になる。CHART-A4 でドーナツを使うなら `layout: "horizontal"`
   を指定するか、CHART-A1 を選ぶ。

## CHART-A1〜A4 (情報レイアウト) と CHART-XX (描画) の独立性

スライドのデザインを変えたいとき (例: 「右テキストを 5 観点に拡張したい」)
は **テンプレ側 (CHART-A2 など)** だけ触る。
チャートの描画品質を変えたいとき (例: 「縦棒のデフォルト色を変えたい」)
は **チャート側 (CHART-XX) と _chart-style.js** だけ触る。両者を切り離して
運用することで、片方の変更が片方を壊さない設計にしてある。
