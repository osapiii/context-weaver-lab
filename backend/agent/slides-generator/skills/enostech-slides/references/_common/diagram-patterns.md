## 色使いのルール

各パターンで `tokens.js` の `diagramPalette` 6 トラック（`navy` / `teal` / `purp` / `gold` / `sage` / `rose`）を **要素の区別に使う**。

### 基本原則

- **3 要素以下**: 1 色（または 2 色）で十分
- **4–6 要素**: `T.diagramTrack(0)` から順番に割り当て（navy → teal → purp → gold → sage → rose）
- **ハイライトが必要**: 目立たせたい要素には `purp`（ブランド色と親和）を当てる

### 例

```javascript
const T = require('../assets/tokens');
const D = T.diagramPalette;

// 4 段階のステップアップ図 — 順番にトラックを取り出す
steps.forEach((st, i) => {
  const pal = T.diagramTrack(i);  // navy / teal / purp / gold ...
  // pal.deep = 濃色（文字・主張）
  // pal.mid  = 中色（塗り本体）
  // pal.bg   = 淡色（背景・ハロー）
});

// プロット図で自社を強調
const items = [
  { code: '自社', pal: D.purp, highlight: true },   // ハロー付き
  { code: 'A',    pal: D.navy },                    // 通常
  { code: 'B',    pal: D.rose },
];
```

## マッキンゼー品質の 5 原則

ダイアグラムを描く時、常に意識するチェックリスト：

1. **コンテンツエリアを厳守** — `y: 1.60 〜 5.10`, `x: 0.40 〜 9.60`。はみ出しゼロ
2. **1 スライド 2–3 色まで** — 6 トラック全部は使わない。絞ると高級感が出る
3. **淡色フィル + 濃色文字** — 暗い塗りで文字を白抜きするのは最小限。軽やかさを優先
4. **余白を恐れない** — 要素を詰め込まない。情報密度よりも読みやすさ
5. **必ず主張を添える** — プロット図には INSIGHT パネル、フロー図には下部注釈など、絵だけで終わらせない

## 実装の流れ

1. この `diagram-patterns.md` で該当パターンを選ぶ
2. `scripts/diagram-patterns.js` の該当ブロック（`// ── 02 サイクル図 ──` 等のコメントで区切られている）をコピー
3. 原子要素（データドット、4象限背景、軸、カード等）を使って組み立てる — 詳細は `references/_common/diagram-expression-patterns.md`
4. トークンは `assets/tokens.js` の `diagramPalette` / `diagramSize` から取る

## DIAG-10〜19 の実装場所

DIAG-10〜19 は `scripts/diagram-patterns.js` 内に以下の構造で実装:

- 共通ヘルパー: `_check` / `_drawLabel` / `_drawArrow` / `_drawDashedBox`
- 描画本体: `_drawCompare2colArch` / `_drawDataflow3stage` / `_drawStack3layer` /
  `_drawTradeoffBalance` / `_drawDecisionTree` / `_drawBizgram3x3` / `_drawLogicTree` /
  `_drawOrgHierarchy` / `_drawCustomerJourney` / `_drawConceptMap`
- スライドブロック: 「10 データフロー」〜「19 コンセプトマップ」

デッキで使う時は該当 slide block をコピーして `addChrome` / `addTitleBlock` を含めるか、
`_drawXxx(slide, area, h_, payload)` だけ抜き出して任意の area に描画する。
